# mood_player → src 통합 계획

## Context (왜 이 작업을 하는가)

이 저장소에는 위치 기반 무드 음악 플레이어가 **두 벌** 존재한다.

- **`src/`** — Vue 3 앱의 완성된 **디자인/와꾸**. 플레이어 바(`PlayerBar`), 재생 시트(`NowPlayingSheet`),
  무드 카드(`MoodCard`), 리액티브 상태머신(`useCrossfadePlayer`)까지 다 만들어져 있으나 **소리가 안 나고
  무드 추론도 없다.** 오디오 엔진(`lib/engines.js`)은 전부 no-op 스텁, 무드는 `useMoodPlaceholder`로
  수동 선택만, `config/moods.js`는 합성음 레시피 **플레이스홀더**다.
- **`mood_player/`** — 실제로 **동작하는 로직**. `moodEngine.js`(LLM 주 + 규칙 폴백 + 캐시 무드추론),
  `crossfadePlayer.js`(실제 `<audio>` 2채널 크로스페이드), `moodConfig.js` + 최상단 `curation.json`
  (실제 R2 호스팅 mp3 트랙).

목표: **로직은 mood_player, 디자인·와꾸는 src를 유지**하여, src의 UI 컴포넌트/상태머신은 그대로 두고
그 안의 "빈 껍데기" 세 지점(무드추론·오디오출력·트랙데이터)만 mood_player의 검증된 로직으로 채운다.
결과적으로 src 앱이 실제로 위치→무드 자동추론→실제 음악 크로스페이드 재생까지 동작한다.

### 확정된 결정 (사용자 확인 완료)
1. **무드 체계**: mood_player 것(`calm/muse/flutter/vivid/savory/dreamy`)을 표준으로 채택.
   src의 색·accent·설명(디자인 토큰)은 가장 가까운 무드로 매핑해 유지, 라벨/​id만 mood_player로.
2. **음원 토글 제거**: 실제 mp3 단일 소스. NowPlayingSheet의 '생성 앰비언트/유튜브' 토글·안내문 삭제.
3. **검증**: `npm run dev`로 앱을 띄워 무드 자동추론→색 전환→실제 mp3 크로스페이드 재생까지 확인.

## 핵심 계약 (integration seam)

src `useCrossfadePlayer`가 UI에 노출하는 player 인터페이스는 **변경하지 않는다**:
`{ isUnlocked, isPlaying, isCrossfading, volume, currentTrack, playlist, progress, remainingSec,
error, toggle, next, prev, isRepeat, toggleRepeat }`. 트랙 shape = `{ id, title, url }`.

`useCrossfadePlayer`가 기대하는 엔진 덱 인터페이스:
`init() · loadOnDeck(deck, track) · fadeDeck(deck, target, sec) · stopDeck(deck) ·
setMasterVolume(v) · suspend() · resume() · dispose()` → 이 인터페이스 뒤에 실제 오디오를 구현한다.

무드 id 매핑 (색·accent·설명 출처가 될 가장 가까운 src 무드):
`calm←serene · muse←contemplative · vivid←vital · flutter←festive · savory←gourmet · dreamy←citywalk`

## 변경 파일 및 작업

### 1. `src/config/moods.js` — 실제 무드 + 트랙으로 재작성
- `MOODS` 배열을 6개 실제 무드로 교체: id/label = mood_player(`calm 고요, muse 사색, flutter 설렘,
  vivid 활기, savory 미식, dreamy 몽환`). `color`는 mood_player 캐논색 사용,
  `accent`·`description`은 위 매핑의 src 무드 것을 유지(라벨 의미에 맞게 dreamy 등은 문구 소폭 조정).
- 각 무드 `tracks`: 최상단 `curation.json`의 트랙을 이식 → `{ id: '<mood>-<n>', title, url: R2_BASE + '/' + r2Key }`.
  `R2_BASE = 'https://pub-3fc25dd84d8b400c8e7c624382630552.r2.dev'` 상수로. (`recipe`·`youtubeIds` 필드 제거)
- `contentTypeIds`는 규칙 폴백용으로 유지하고, 규칙 엔진이 쓸 매핑을 함께 export:
  `TYPE_TO_MOOD = {12:'calm',25:'calm',14:'muse',15:'flutter',28:'vivid',38:'vivid',39:'savory',32:'dreamy'}`
  및 `TYPE_LABEL`(또는 `config/contentTypes.js`의 라벨 재사용).
- `DEFAULT_MOOD_ID = 'calm'`. `MOOD_BY_ID`, `moodById()`는 그대로 유지.

### 2. `src/lib/moodEngine.js` — 신규 (mood_player/vue/moodEngine.js 이식)
- `mood_player/vue/moodEngine.js`를 src로 복사. `haversine/nearby/kstHour/isNight/ruleMood/promptText/
  llmMood/signature/inferMood` 그대로. 핵심 로직(LLM gpt-5-mini + strict JSON schema, `CONF=0.45`,
  캐시, 절대 throw 안 함)은 손대지 않는다.
- import만 src에 맞게: `TYPE_TO_MOOD`/라벨을 `../config/moods`(+ contentTypes)에서,
  `ruleMood` 안 `MOODS[best].label`은 `moodById(best).label`로 조정.

### 3. `src/lib/engines.js` — 실제 오디오 엔진으로 재작성
- no-op `AmbientEngine`을 **실제 2-덱 오디오 엔진**으로 구현 (`crossfadePlayer.js`의 볼륨 램프 로직 이식):
  덱별 `new Audio()` 2개, `loadOnDeck`=`src` 지정+`volume=0`+`play().catch()`,
  `fadeDeck`=setInterval 볼륨 램프(master 반영), `stopDeck`=pause+volume 0,
  `setMasterVolume`/`suspend`/`resume`/`dispose`.
- **중요(자동재생 정책)**: `init()`은 동기·경량으로 유지해 최초 `play()`가 사용자 제스처 컨텍스트를
  잃지 않게 한다(`await` 뒤로 play가 밀리지 않도록). CORS 불필요(element.volume 방식).
- `audio.loop = true`로 단일 트랙 무드가 중간에 끊기지 않게. `YouTubeEngine`은 삭제.

### 4. `src/composables/useCrossfadePlayer.js` — 유튜브/소스 분기 제거
- `source` ref, `playlist`의 youtube 분기, `createEngine`의 youtube 경로, `watch(source)` 삭제.
- `playlist = mood.tracks`로 단순화. `createEngine()`은 실제 오디오 엔진만 반환.
- 나머지(90s intervalMs 오토어드밴스, repeat, moodId watch 크로스페이드, unlock)는 유지.

### 5. `src/composables/useMood.js` — 신규 (useMoodPlaceholder 대체, 실제 추론)
- `useMood(places)` 반환 `{ moodId, mood, setMood, moods, moodInfo }`.
- `watch(places, async list => { if(!list.length) return; const r = await inferMood(adapt(list),
  kstHour(), import.meta.env.VITE_OPENAI_API_KEY); moodId.value = r.moodId; moodInfo.value = r })`
  (`immediate:true`). 키 없으면 엔진이 규칙 폴백으로 동작(정상).
- `adapt(list)`: src place → 엔진 입력 `{ title, typeId, dist: place.distanceM, type: place.type.label }`.
- `setMood(id)` 수동 오버라이드 유지(MoodCard 피커가 계속 동작).
- `src/composables/useMoodPlaceholder.js`는 제거(또는 이 파일로 대체).

### 6. `src/App.vue` — 무드 컴포저블 연결 변경
- import를 `useMood`로 교체, `places`를 얻은 뒤 `useMood(places)` 호출(현재도 places가 mood보다 먼저 생성됨).
- `provide('app', {...})`에 `moodInfo` 추가(선택: "왜 이 무드인지" 표시용). 나머지 셸/도크/테마 로직 불변.

### 7. `src/lib/pois.js` — typeId 노출
- 로컬 POI 매핑에 `typeId: Number(it.contenttypeid)` 추가(규칙 폴백·프롬프트가 쓰도록).
  (카카오 경로는 typeId 없음 → LLM은 카테고리 라벨로, 규칙은 기본 calm 폴백. 허용 범위, 주석으로 명시.)

### 8. `src/data/placeholder.js` — 커뮤니티 더미 moodId 재매핑
- `CURATIONS[].moodId` 및 `MOOD_TRACKS` 키를 신 id로 변환:
  `serene→calm, contemplative→muse, vital→vivid, festive→flutter, gourmet→savory, citywalk→dreamy`.
  (그래야 MoodCard "다른 곡", CommunityView/CurationDetail의 `moodById(moodId)` 라벨·색이 맞음)

### 9. `src/components/NowPlayingSheet.vue` — 음원 토글 제거
- `SOURCES` 배열, 음원 radiogroup, 유튜브/앰비언트 안내문(`hint`), `p.source` 참조 삭제. 볼륨 슬라이더는 유지.
- (선택) `moodInfo.reason`/`decidedBy` 노출로 "왜 이 무드인지" 표시.

### 10. `src/components/MoodCard.vue` — (선택) 플레이스홀더 배지 제거
- 이제 실제 무드이므로 `플레이스홀더` 배지 문구 정리. 피커는 수동 오버라이드로 유지.

### 잔여 점검
- src 전역에서 구 무드 id(`serene/contemplative/vital/festive/gourmet/citywalk`) 잔존 참조 grep 후 정리
  (특히 `CurationNewView.vue`의 무드 선택 옵션).
- `.env`에 `VITE_OPENAI_API_KEY` 존재 확인(있으면 LLM 추론, 없으면 규칙 폴백). 키 이름은 챗봇과 동일해 신규 설정 불필요.

## 재사용할 기존 자산
- 추론: 이식된 `moodEngine.inferMood/ruleMood/kstHour` (mood_player 검증 로직 그대로)
- 트랙 데이터: `curation.json`(제목·r2Key·출처 — 라이선스 명세 요건에도 부합)
- 근접 장소: `useNearbyPlaces`가 이미 `distanceM`으로 정렬해 제공 → 엔진 `nearby()` 재호출 불필요
- 무드 조회: `config/moods.js`의 `moodById`, `TYPE_TO_MOOD`
- 엔진 인터페이스: `useCrossfadePlayer`의 덱 계약 (변경 없음)

## 검증 (end-to-end)
1. `npm run dev` 실행 후 브라우저 접속.
2. 위치 권한 허용/폴백(서울시청) → 근처 장소 로드 확인.
3. **무드 자동 추론**: 상단 브랜드 점·플레이어 바·카드 색이 추론된 무드색으로 전환되는지(`--mood` CSS 변수).
   OpenAI 키 있으면 LLM, 없으면 규칙 폴백(콘솔/`moodInfo.decidedBy`로 확인).
4. **실제 재생**: 재생 버튼(사용자 제스처) 클릭 → R2 mp3 소리 재생, 90초 후 크로스페이드로 다음 트랙 전환,
   prev/next/일시정지/볼륨 동작 확인. 다중 트랙 무드(flutter/vivid/dreamy)에서 곡 인덱스(`n/of`) 변화 확인.
5. MoodCard 무드 칩 수동 선택 시 즉시 크로스페이드로 해당 무드 첫 곡 전환 확인.
6. `npm run build` 통과(합성음/유튜브 잔재 import 오류 없음) 확인.
