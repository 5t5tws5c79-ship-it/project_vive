# 무드 플레이어 — 컴포넌트 명세 (엔진 + 파이프라인 일체형)

> `feat_audio_player.md`(초기 피처 플랜)를 이번 논의 기준으로 고도화한 문서.
> 초기 플랜을 대체하며, 관점이 "앱 기능"에서 "임베드 컴포넌트 경계"로 바뀌었다.

## 0. 담당 경계

- **내 담당**: 위치 → 근접 장소 → 무드 추론 → 플레이리스트 → 크로스페이드 재생까지의 **전 파이프라인을 감싼 재사용 컴포넌트 1개 + 헤드리스 엔진(composable)**.
- **FE(타 담당)**: 화면 레이아웃의 플레이스홀더에 컴포넌트를 장착하고, props로 데이터·키를 주입. 플레이어 자체는 만들지 않는다.
- 즉 산출물은 "돌아가는 화면"이 아니라 **드롭인 가능한 컴포넌트 폴더**다. (§5 계약이 이 문서의 핵심)

## 1. 이번 논의로 확정된 설계 방향

1. **무드 결정 = LLM 의미 추론(주) + 규칙표(폴백).**
   근처 장소의 *이름·조합*을 OpenAI가 읽어 무드를 고른다. 초기 플랜처럼 `contentTypeId → 무드`를 표로 못 박지 않는다. 표는 LLM 실패 시 폴백일 뿐.
2. **무드 체계 = 분위기 우선 정의 → 다대일 매핑.**
   무드를 데이터 스키마에서 역산하지 않는다. 분위기(고요/사색/…)를 먼저 정의하고, 여러 장소·시간대가 하나의 무드로 모이게 한다.

> 초기 플랜의 문제였던 점: F4의 하드코딩 표가 무드를 이미 결정해버려서 F3의 LLM이 잉여였고, 6무드가 6개 `contentTypeId`에 1:1로 붙어 스키마 역산이었다. 위 두 결정으로 둘 다 해소.

## 2. 아키텍처 (레이어)

프레임워크에 안 묶인 **엔진(순수 TS)** 위에 얇은 Vue3 래퍼를 얹는다. 엔진이 Vue 반응성에 안 엉켜야 테스트·재사용이 쉽다.

```
engine/            # 프레임워크 무관 (순수 TS)
  location.ts      # 현재 위치 획득 + 폴백
  nearby.ts        # 하버사인 근접 상위 N
  mood.ts          # 무드 추론: LLM 주 + 규칙 폴백 + 캐시
  playlist.ts      # 무드 → 트랙 목록
  player.ts        # 크로스페이드 플레이어 (Web Audio A/B)
  sources/         # generative / youtube 어댑터
useMoodPlayer.ts   # 엔진 ↔ 반응형 상태 배선 (헤드리스)
MoodPlayer.vue     # 위 composable을 쓰는 기본 UI 컴포넌트 (드롭인)
config.ts          # MOODS 정의, 파라미터 기본값
types.ts
```

## 3. 무드 체계 (분위기 우선 · 다대일)

무드는 `config.ts`의 `MOODS`로 교체 가능. 아래는 초기 6종 제안.

| moodId | 라벨 | 분위기 결 | 대표 장소(예) | 폴백 규칙 매핑(contentTypeId) |
|--------|------|-----------|----------------|-------------------------------|
| calm | 고요 | 잔잔·정적 | 산·사찰·수목원 | 12 관광지, 25 여행코스 |
| muse | 사색 | 차분·내면 | 미술관·박물관·고궁 | 14 문화시설 |
| flutter | 설렘 | 들뜸·기대 | 축제·핫플 | 15 축제공연행사 |
| vivid | 활기 | 역동·번화 | 레포츠·시장·번화가 | 28 레포츠, 38 쇼핑 |
| savory | 미식 | 아늑·풍미 | 맛집·카페 골목 | 39 음식점 |
| dreamy | 몽환 | 야경·드라이브 | 해안·야경 스팟 | (폴백엔 없음 — 주로 LLM/시간대) |

- **다대일**: 예) `활기 ← {레포츠, 번화가 쇼핑, 시장}`.
- `몽환`은 카테고리로 안 잡히는 결(시간대·야경)이라 폴백 규칙에선 `고요`로 흡수되고, 주로 LLM/시간 맥락에서만 등장. → 규칙 폴백만 돌 때는 사실상 5무드.

## 4. 무드 추론 (LLM 주 + 규칙 폴백)

**입력**: 근처 상위 N개 장소 `[{name, categoryLabel, distanceM}]` + 선택 맥락 `{hour, weather?}`.
**LLM 출력(구조화 강제)**:
```json
{ "moodId": "calm|muse|flutter|vivid|savory|dreamy",
  "confidence": 0.0,
  "reason": "왜 이 무드인지 한 문장" }
```
- OpenAI structured output(json_schema / function calling)으로 enum 강제. 무효 JSON·저신뢰(confidence 낮음)·호출 실패 → 규칙 폴백.
- **규칙 폴백**: 근처 장소 `contentTypeId` 최빈값 → §3 표로 무드. (초기 플랜의 규칙을 폴백으로 강등)

**비용 가드 (의뢰서 "예산 초과 금지 / 사용량 제한 키" 직결 — 반드시 반영)**
- 위치가 조금 흔들릴 때마다 호출하면 예산이 샌다. → **호출 트리거를 "위치"가 아니라 "근접 장소 집합의 변화"로 잡는다.**
- 좌표를 격자(예: 소수 3자리)로 반올림 → 상위 N 장소 집합의 서명(signature) 생성 → 서명이 바뀔 때만 LLM 호출. 동일 서명은 캐시 반환.
- 디바운스(예: 이동 후 2~3초 정착 시 1회).

**출력**: `{moodId, label, reason, decidedBy: 'llm'|'rule', confidence}` — 화면에 "왜 이 무드인지" 설명 가능하게 유지.

## 5. 컴포넌트 계약 (FE가 보는 전부)

### Props (입력)
| prop | 타입 | 기본 | 설명 |
|------|------|------|------|
| `poiData` | `Poi[]` | — | 제공 JSON(권역 선정분). FE가 주입 |
| `location` | `{lat,lng,source}?` | `undefined` | 있으면 사용, 없으면 컴포넌트가 geolocation 시도 |
| `moods` | `MoodConfig[]?` | 내장 6종 | 무드·색·설명·사운드 레시피·youtube ID 교체 |
| `sourceMode` | `'generative'\|'youtube'` | `'generative'` | 오디오 소스 |
| `nearbyCount` | `number` | `5` | 근접 상위 N |
| `transitionMs` | `number` | `90000` | 트랙 전환 간격 |
| `fadeMs` | `number` | `4000` | 크로스페이드 길이 |
| `moodOverride` | `moodId?` | `undefined` | 무드 강제(수동 선택 대응) |
| `autoStart` | `boolean` | `false` | 자동재생 정책상 기본 false |

> OpenAI 키는 prop로 받지 않고 `import.meta.env.VITE_OPENAI_API_KEY` 사용 (의뢰서 §챗봇 규약과 동일 키·정책).

### Emits (출력)
| event | payload | 시점 |
|-------|---------|------|
| `mood-changed` | `{moodId,label,reason,decidedBy,confidence}` | 무드 확정 |
| `track-changed` | `{track,index}` | 트랙 전환 |
| `state-changed` | `{playing,paused}` | 재생 상태 |
| `transition-progress` | `{remainingMs,ratio}` | 다음 전환 카운트다운 |
| `location-resolved` | `{lat,lng,source}` | 위치 확정 |
| `error` | `{stage,message}` | 각 단계 실패 |

### 노출 메서드 (`ref`/`defineExpose`)
`play()` · `pause()` · `next()` · `setVolume(v)` · `setMood(id)` · `refreshLocation()`

## 6. 크로스페이드 엔진 & 소스 모드

- **엔진**: Web Audio `GainNode` 2채널(A/B)을 `AudioContext.currentTime` 기준 램프로 오버랩 전환. `fadeMs` 동안 A 페이드아웃 ↔ B 페이드인.
- **generative(기본)**: 무드별 사운드 레시피로 Web Audio가 직접 앰비언트 생성. 외부 의존·라이선스 리스크 0 → **기본값 권장.**
- **youtube(선택)**: 숨김 플레이어 2개 `setVolume` 크로스페이드. 무드별 영상 ID 필요. 단 **임베드 저작권/차단 리스크**가 있어 데모 안전판은 generative.
- 자동재생 정책: 첫 재생은 사용자 제스처("시작")로 언락. `autoStart` 기본 false.

## 7. 확정 필요 — 경계 결정 3가지

아래는 내가 기본값을 정해뒀지만 **당신 확인이 필요한 API 형태 결정**이다.

1. **산출물 형태**
   - (권장) **헤드리스 `useMoodPlayer` composable + 기본 `MoodPlayer.vue` UI 둘 다** 제공. FE는 `<MoodPlayer/>`를 꽂거나, 커스텀 UI를 원하면 composable을 씀.
   - 대안 A: UI 포함 위젯만 (FE는 위치만). / 대안 B: 헤드리스만 (FE가 UI 전부).
2. **위치·POI 소유**
   - (권장) geolocation은 **컴포넌트 내부**에서 처리하되 `location` prop로 override 허용. POI JSON은 **FE가 prop 주입**(제공 JSON 로딩은 FE 몫이므로 결합도↓).
3. **오디오 기본 소스**
   - (권장) `generative` 기본 + `youtube` 옵션. (라이선스·차단 리스크 회피)

## 8. 의뢰서 제약 정합성 체크

| 제약 | 이 컴포넌트에서 지키는 법 |
|------|--------------------------|
| Vue3 SPA / 백엔드 금지 | 전 과정 클라이언트에서 동작. 서버 없음 |
| OpenAI 키 `VITE_`·사용량 제한·예산 초과 금지 | env 키 사용 + §4 비용 가드(집합 변화 시에만 호출·캐시) |
| 추가 데이터 라이선스 | POI는 제공 JSON만. 오디오는 generative 기본으로 저작권 회피 |
| `.env` 미포함 | 키는 env로만, 커밋 제외 |
| 착수 후 범위 변경 불가 | §7 확정 후 계약 동결하고 시작 |
